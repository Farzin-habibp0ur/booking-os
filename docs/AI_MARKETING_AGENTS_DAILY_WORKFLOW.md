# AI Marketing Agents — Operator Daily Workflow (In-App Agents)

> **Scope:** This document covers the **12 in-app marketing agents** that run inside the NestJS API for each customer business. These are code-based agents in `apps/api/src/modules/agent-framework/` managed via `/marketing/agents` and `/marketing/queue` pages.
>
> **Not covered here:** BookingOS's own internal growth engine (15 prompt-based agents in `agents/` directory). For that system, see the agent prompt files and `system/quality-gates.md`.

> A step-by-step guide for BookingOS operators to manage the 12 AI marketing agents throughout the day. Each step specifies **what to do**, **where to do it** (exact page/route), and **why it matters**.

---

## The 12 Marketing Agents at a Glance

| Category | Agent | What It Does |
|----------|-------|-------------|
| **Content** | BlogWriter | Drafts blog posts across 5 content pillars |
| **Content** | SocialCreator | Creates social media posts for multiple channels |
| **Content** | EmailComposer | Writes marketing emails and campaign copy |
| **Content** | CaseStudy | Produces customer success stories |
| **Content** | VideoScript | Writes video scripts and storyboards |
| **Content** | Newsletter | Assembles newsletter editions |

| **Distribution** | ContentPublisher | Publishes approved content to channels |

| **Analytics** | TrendAnalyzer | Identifies trending topics and opportunities |

## Step-by-Step Workflow

### ① Morning Briefing Review

**Where:** Dashboard → Briefing Feed (top of main dashboard)
**Route:** `/dashboard`
**When:** First thing in the morning

The Dashboard renders a **BriefingFeed** above the metric cards. This is generated by the `OpportunityDetectorService` and groups ActionCards by urgency. The feed auto-refreshes every 5 minutes.

**What to do:**

1. Open the Dashboard at `/dashboard`.
2. Review the **BriefingFeed** at the top of the page. Cards are color-coded by priority:
   - 🔴 **URGENT_TODAY** (red) — Time-sensitive items that need immediate action.
   - 🟡 **NEEDS_APPROVAL** (amber/lavender) — Content or actions waiting for your sign-off.
   - 🟢 **OPPORTUNITY** (sage) — Suggestions the agents have identified (e.g., trending topics, open slots to promote).
   - ⚪ **HYGIENE** (slate) — Maintenance tasks (e.g., stale drafts, expiring offers).
3. For each card, choose an action:
   - **Approve** — Accept the agent's recommendation.
   - **Dismiss** — Reject it (with optional reason).
   - **Snooze** — Defer it using the snooze dropdown (1 hour, 4 hours, tomorrow, next week).
4. Expand any card for more details by clicking on it. Each card type has a contextual CTA label (e.g., "Review Draft", "Approve Campaign", "Check Metrics").

**Why:** This gives you the 30-second pulse on everything your marketing agents did overnight and what needs attention right now.

---

### ② Action Card Triage

**Where:** AI Command Center → Actions tab
**Route:** `/ai/actions`
**When:** Immediately after the briefing review

The Actions tab in the AI Command Center provides a comprehensive view of **all** pending action cards, not just the briefing highlights.

**What to do:**

1. Navigate to `/ai/actions`.
2. Review pending action cards grouped into the 4 categories (URGENT_TODAY, NEEDS_APPROVAL, OPPORTUNITY, HYGIENE).
3. Use **bulk actions** to efficiently process multiple cards:
   - Select multiple cards of the same type.
   - Bulk-approve content that looks good.
   - Bulk-dismiss irrelevant suggestions.
4. Pay special attention to NEEDS_APPROVAL cards — these are content drafts or distribution plans the agents created but need human sign-off before going live.
5. Cards have a lifecycle: `PENDING → APPROVED / DISMISSED / SNOOZED / EXECUTED / EXPIRED`. Expired cards auto-close via a background cron job.

**Why:** This is your central command for approving or rejecting everything the agents want to do. Keeping this queue clear ensures agents aren't blocked.

---

### ③ Content Approval Queue

**Where:** Marketing → Content Queue
**Route:** `/marketing/queue`
**When:** Morning, after action card triage

This is the dedicated content review interface with 9 API endpoints for granular control over drafts.

**What to do:**

1. Navigate to `/marketing/queue`.
2. Review the queue of **ContentDraft** items. Each draft includes:
   - **Content type** — One of 6 types: Blog, Social, Email, CaseStudy, VideoScript, Newsletter.
   - **Channel** — Where it will be published (Website, Instagram, Facebook, Email, LinkedIn, WhatsApp).
   - **Content pillar** — Which strategic pillar it serves (Industry Insights, Product Education, Customer Success, Thought Leadership, Technical).
   - **Status** — DRAFT, REVIEW, APPROVED, or PUBLISHED.
3. For each content piece in REVIEW status:
   - Read the draft content.
   - **Approve** it to move it to the APPROVED state (distribution agents will pick it up).
   - **Reject** it with feedback so the content agent can revise.
   - **Edit** it directly if only minor changes are needed.
4. Use **bulk-approve** or **bulk-reject** endpoints for efficiency when reviewing multiple similar items.
5. Check content pillar distribution — aim for balanced coverage across all 5 pillars over time.

**Why:** This is the quality gate. Nothing gets published without passing through here (unless autonomy is set to FULL_AUTO). This is where you shape the brand voice and ensure accuracy.

### ④ Agent Status & Health Check

**Where:** Marketing → Agents Dashboard **and** AI Command Center → Agents tab
**Routes:** `/marketing/agents` and `/ai/agents`
**When:** Midday check-in

**What to do:**

1. Go to `/marketing/agents` for the marketing-specific view:
   - Use **tab filters** to switch between Content / Distribution / Analytics agent groups.
   - Check that all agents you expect to be active have their **toggle enabled**.
   - Review the last run time for each agent — if an agent hasn't run when expected, investigate.
   - Use the **"Run Now"** button to manually trigger any agent that needs an immediate run (e.g., if you just approved a batch of content and want the ContentScheduler to plan distribution now).

2. Go to `/ai/agents` for the broader AI Command Center view:
   - This shows **all agents** (both the 5 operational agents and 12 marketing agents).
   - Each agent has an expandable **run history** — click to see recent runs, outcomes, and any errors.
   - Check for any agents in an error or stalled state.

**Why:** Agents run autonomously on schedules, but things can go wrong — API rate limits, content generation failures, queue backlogs. A midday check catches issues before they compound.

---

### ⑤ Email Sequence Monitoring

**Where:** Marketing → Email Sequences
**Route:** `/marketing/sequences`
**When:** Midday, as part of the health check

BookingOS comes with 7 default drip campaigns: Welcome, FeatureEducation, SocialProof, TrialExpiry, WinBack, Upgrade, and Referral.

**What to do:**

1. Navigate to `/marketing/sequences`.
2. Review the stats for each active sequence (open rates, click rates, conversion rates).
3. Check that sequences are **toggled on/off** appropriately:
   - Ensure the Welcome sequence is active if you're onboarding new customers.
   - Enable WinBack for re-engaging lapsed clients.
   - Pause any sequences that aren't relevant to your current goals.
4. **Expand the timeline** for any sequence to see where customers are in the drip flow and whether there are bottlenecks (e.g., high drop-off at a specific step).
5. If the EmailComposer agent has suggested new email variants or A/B tests, review them here.

**Why:** Email sequences are a major revenue driver for service businesses. Monitoring ensures customers are getting the right messages at the right time.

### ⑥ Performance Review

**Where:** AI Command Center → Performance tab
**Route:** `/ai/performance`
**When:** Afternoon

**What to do:**

1. Navigate to `/ai/performance`.
2. Review the **KPI cards** at the top for headline metrics (content output, engagement rates, conversion, ROI).
3. Use the **date range selector** to compare this week vs. last week or this month vs. last month.
4. Examine the **Recharts visualizations** for trends:
   - Content production volume over time.
   - Engagement rates by channel and content type.
   - Agent efficiency (successful runs vs. errors).
5. Review the **staff feedback summary** — this aggregates thumbs up/down ratings from your team on agent outputs.
6. Identify underperforming agents or content types and note them for settings adjustments.

**Why:** Data-driven decisions. If the SocialCreator's Instagram posts are outperforming blog content, you might want to increase its run frequency or shift content pillar emphasis.

### ⑦ Agent Feedback & Coaching

**Where:** AI Command Center → Agents tab (feedback on individual runs)
**Route:** `/ai/agents` → expand an agent → run history → feedback button
**API:** `/agent-feedback`
**When:** Afternoon, after performance review

**What to do:**

1. Go to `/ai/agents` and expand the run history for agents whose output you reviewed today.
2. For each run you evaluated (either through the content queue or action cards):
   - Give a **thumbs up** 👍 if the output was good and on-brand.
   - Give a **thumbs down** 👎 if the output missed the mark.
   - Add a **written comment** explaining what was good or what needs improvement (e.g., "Tone was too formal for our Instagram audience" or "Great use of customer data in the case study").
3. Review the **aggregation stats** to see feedback trends over time — are agents improving based on your guidance?

**Why:** Feedback is how the agents learn your preferences. Consistent feedback trains the system to produce better outputs over time. The aggregation stats help you see whether your coaching is working.

### ⑧ Settings & Autonomy Tuning (As Needed)

**Where:** Settings → Agents and Settings → Autonomy
**Routes:** `/settings/agents` and `/settings/autonomy`
**When:** As needed — when you want to change how agents behave

#### Agent Settings (`/settings/agents`)

1. Navigate to `/settings/agents`.
2. For each agent, you can configure:
   - **Enable/Disable** — Turn individual agents on or off.
   - **Schedule** — Set the run interval (e.g., BlogWriter every 24 hours, SocialCreator every 6 hours).
   - **Autonomy level** — How much freedom the agent has (see below).
   - **Performance summary** — Quick view of the agent's recent track record.

#### Autonomy Settings (`/settings/autonomy`)

1. Navigate to `/settings/autonomy`.
2. Configure per-action-type autonomy levels. The four levels are:

| Level | Behavior |
|-------|----------|
| **OFF** | Agent is disabled for this action type. |
| **SUGGEST** | Agent creates drafts/suggestions but takes no action. Everything requires manual approval. |
| **AUTO_WITH_REVIEW** | Agent acts autonomously but flags actions for post-hoc review. You can reverse actions after the fact. |
| **FULL_AUTO** | Agent acts completely independently. No approval required. |

3. **Recommended starting configuration:**
   - Content creation agents → **SUGGEST** (you review all content before it goes out).
   - ContentScheduler → **AUTO_WITH_REVIEW** (let it plan, but review the schedule).
   - ContentPublisher → **SUGGEST** (keep publishing gated until you trust the pipeline).
   - Analytics agents → **FULL_AUTO** (these only analyze, they don't publish anything).

4. As you build confidence in specific agents, gradually increase their autonomy level from SUGGEST → AUTO_WITH_REVIEW → FULL_AUTO.

**Why:** This is how you calibrate the human-AI balance. Start conservative, increase autonomy as trust builds. The per-action-type granularity means you can let analytics run freely while keeping content creation under tight review.

### ⑨ Monthly Review (End of Month)

**Where:** Reports → Monthly Review
**Route:** `/reports/monthly-review`
**When:** End of each month

**What to do:**

1. Navigate to `/reports/monthly-review`.
2. Review the **AI-generated monthly summary** which includes:
   - Total content pieces produced, approved, published, and rejected.
   - Content performance by pillar and channel.
   - Action card stats (how many generated, approved, dismissed, expired).
   - Agent run stats (total runs, success rate, error rate).
   - ROI metrics from the ContentROI agent.
3. Use these insights to adjust your strategy for the next month:
   - Shift content pillar emphasis if one is underperforming.
   - Adjust agent schedules based on what's working.
   - Update autonomy levels based on agent track records.

**Why:** The monthly review closes the feedback loop at the strategic level. It's where you step back from daily operations and assess whether the AI marketing system is delivering business value.

## Tips for Effective Agent Management

1. **Start conservative.** Set all content agents to SUGGEST mode. Only increase autonomy after 2–3 weeks of consistent, high-quality output.

2. **Be specific in feedback.** "Bad" tells the agent nothing. "The tone was too casual for a B2B audience and the CTA was buried" helps it improve.

3. **Check the content pillar balance weekly.** If all your content is "Product Education" and nothing is "Thought Leadership," adjust agent prompts or pillar priorities in settings.

4. **Use "Run Now" strategically.** After approving a batch of blog posts, trigger the ContentScheduler immediately so distribution planning starts right away instead of waiting for the next scheduled run.

5. **Don't ignore analytics agents.** TrendAnalyzer and ContentROI generate action cards with insights. Dismissing these without reading them means missing optimization opportunities.

6. **Review the AI Command Center Overview weekly.** The `/ai` overview page shows the activity feed and autonomy levels at a glance — useful for spotting patterns you might miss in daily triage.

7. **Pair the monthly review with settings changes.** The best time to adjust agent schedules, autonomy levels, and content strategies is right after reviewing the monthly data.
